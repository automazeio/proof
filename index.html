Capture Modes

Three ways to capture proof.

Pick the one that matches your test type. Mix and match in a single run.

Terminal

Runs any shell command. Captures stdout/stderr with real timestamps into a self-contained HTML player.

Any runner 
$ proof capture \
  --app my-app \
  --command "pytest tests/ -v" \
  --mode terminal

Works with pytest, go test, cargo test, bun test, make check — anything that writes to stdout.

Browser

Runs Playwright tests with video recording. Supports device emulation, custom viewports, and cursor highlights.

Playwright 
$ proof capture \
  --app my-app \
  --test-file tests/checkout.spec.ts \
  --mode browser \
  --device "iPhone 14"

Pass an array of devices or viewports to capture the same test across multiple screen sizes in one call.

Simulator

Records iOS Simulator or Android emulator while your XCUITest or Espresso tests run. Overlays pixel-accurate tap indicators automatically.

iOS · Android 
$ proof capture \
  --app my-app \
  --mode simulator \
  --platform ios \
  --device-name "iPhone 17 Pro" \
  --command "xcodebuild test ..."
Learn more about simulator mode →

Mobile & Simulators

Native app testing, with tap indicators.

Proof drives the simulator locally, records the screen, and overlays red dot + ripple tap indicators at the exact position of each tap — so reviewers see what was tapped, not just that tests passed.

Simulators must be installed locally

Proof drives the simulator on your machine — it does not download or provision them. iOS: install via Xcode → Settings → Platforms. Android: create an AVD in Android Studio → Tools → Device Manager.

iOS (XCUITest)
  • → Records via xcrun simctl recordVideo
  • → Add ProofTapLogger.swift to your test target
  • → Replace .tap() with .proofTap()
  • → Pixel-accurate overlays from xcresult + tap log
-parallel-testing-enabled NO
Android (Espresso)
  • → Records via adb emu screenrecord
  • → Works on Apple Silicon (bypasses gfxstream issue)
  • → Write tap coords to $PROOF_TAP_LOG
  • → Auto-boots AVD if not already running
PROOF_TAP_LOG=/tmp/proof-taps.json

Get Started

Install in 10 seconds.

macOS / Linux
$ curl -fsSL https://automaze.io/install/proof | sh
Homebrew
$ brew install automazeio/tap/proof
npm (Node.js / TypeScript)
$ npm install -g @automaze/proof
Quick example
# 1. Capture your tests
proof capture \
  --app my-app \
  --command "npm test" \
  --mode terminal \
  --dir ./evidence

# 2. Generate a report
proof report \
  --app my-app \
  --dir ./evidence \
  --format html

# 3. Attach evidence/my-app/... to your PR

FAQ

Common questions.

Do I need Node.js installed?

No. The CLI is a standalone binary that ships with its own runtime. Install via curl or brew and it works everywhere. The npm package is available as an alternative for projects that already use Node.

Does it work in CI?

Yes. Terminal and browser modes work in any headless CI environment. Simulator mode requires a macOS runner (GitHub Actions macos-latest) with Xcode or Android Studio pre-installed. All CLI output is JSON, making it easy to consume in scripts.

What's the difference between terminal and browser mode?

Terminal mode runs any shell command and captures stdout/stderr as a replayable HTML animation. Browser mode runs a Playwright test file and captures the screen as a video. Use terminal for unit tests, integration tests, or any CLI tool. Use browser for end-to-end UI flows.

How do tap indicators work on iOS vs Android?

On iOS, XCUITest taps are injected at the HID level and don't flow through UIKit — so no runtime overlay can intercept them. Instead, ProofTapLogger.swift logs element coordinates and proof overlays indicators with ffmpeg post-processing. On Android, tap coordinates come from a JSON file your test script writes, and proof overlays them the same way.

Is it free?

Yes. Proof is open source under the Apache 2.0 license. No accounts, no telemetry, no limits.
